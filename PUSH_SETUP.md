# Customer app — FCM push notifications setup

The customer app uses **@react-native-firebase/messaging** (true FCM tokens); the
backend sends via **firebase-admin**. The JS side is wired; what remains is the
Firebase project + a **dev build** (this CANNOT run in Expo Go).

This project uses **Continuous Native Generation** — `android/` and `ios/` are
git-ignored and regenerated by `expo prebuild`. So you configure everything in
`app.json` and regenerate, rather than editing native folders by hand.

## App identifiers (must match the Firebase apps you register)

- Android package: `com.mostafadbagh.jawlahsyreact`
- iOS bundle ID:   `com.mostafadbagh.jawlah-sy-react`  (matches the existing iOS project)

## 1. Firebase project

1. Use the same Firebase project as the backend.
2. **Add an Android app** with package `com.mostafadbagh.jawlahsyreact` →
   download **`google-services.json`** → place at `jawlah_sy_react/google-services.json`.
3. *(iOS)* **Add an iOS app** with bundle ID `com.mostafadbagh.jawlah-sy-react` →
   download **`GoogleService-Info.plist`** → place at
   `jawlah_sy_react/GoogleService-Info.plist`. Then upload your **APNs auth key**
   under Firebase → Cloud Messaging (required for iOS push).
4. The backend's service-account key is separate — see `jawlahapp/.env.example`.

> Both files are git-ignored. Until they exist, `app.json` shows harmless
> "file not found" warnings and prebuild will refuse to run.

## 2. Build a dev client (required — not Expo Go)

```bash
cd jawlah_sy_react
npx expo prebuild --clean      # regenerates android/ + ios/ with Firebase wired
npx expo run:android           # or: npx expo run:ios
# …or EAS: npx eas-cli build --profile development --platform android
```

## 3. Runtime behaviour

- On login the app requests notification permission, gets the FCM token, and
  registers it (`POST /api/v1/saveToken`, stored on `User.fcm_token`). Token
  rotation re-registers automatically.
- Order events (placed / preparing / ready / driver assigned / on the way /
  delivered / cancelled) arrive as push + an in-app notification record.
- Foreground messages show in the snackbar; tapping a notification opens that
  order's details (via `order_id`), or the Orders tab otherwise.
- In Expo Go / web the native module is absent → safe no-op, app runs normally.
